# راهنمای استقرار DocEdit روی Vercel (رایگان)

این راهنما مخصوص اکانت گیت‌هاب **masood5400** نوشته شده. کل مسیر حدود ۱۰ دقیقه طول می‌کشد و همه‌چیز در پلن رایگان انجام می‌شود.

## مرحله ۱ — ساخت ریپازیتوری در گیت‌هاب

ساده‌ترین راه (بدون نیاز به نصب git):

۱. برو به <https://github.com/new>
۲. نام ریپو را بگذار `docedit` (یا هر نامی) و **Create repository** بزن.
۳. در صفحه ریپو روی «uploading an existing file» کلیک کن.
۴. همه فایل‌ها و پوشه‌های همین فولدر را با درگ‌اند‌دراپ آپلود کن (پوشه‌های `api`، `css`، `js` و فایل‌های ریشه). فایل `run-windows.bat` هم اشکالی ندارد باشد.
۵. **Commit changes** بزن.

> اگر git نصب داری، به‌جای آپلود دستی:
> ```
> cd "D:\Resource\document editor"
> git init
> git add .
> git commit -m "DocEdit"
> git branch -M main
> git remote add origin https://github.com/masood5400/docedit.git
> git push -u origin main
> ```

## مرحله ۲ — اتصال به Vercel

۱. برو به <https://vercel.com> و با گزینه **Continue with GitHub** (اکانت masood5400) وارد شو.
۲. **Add New → Project** و ریپوی `docedit` را **Import** کن.
۳. هیچ تنظیم Build لازم نیست (پروژه استاتیک + Serverless Functions است). فقط **Deploy** بزن.

## مرحله ۳ — اتصال دیتابیس Upstash Redis (رایگان)

۱. در داشبورد پروژه در Vercel، تب **Storage** را باز کن.
۲. **Create Database → Upstash → Redis** را انتخاب کن (پلن **Free**).
۳. Connect بزن — Vercel خودش متغیرهای محیطی (`UPSTASH_REDIS_REST_URL` و `UPSTASH_REDIS_REST_TOKEN` یا `KV_REST_API_URL` و `KV_REST_API_TOKEN`) را به پروژه اضافه می‌کند. کد ما هر دو نام را پشتیبانی می‌کند.

## مرحله ۴ — تعیین گذرواژه مدیر

۱. در پروژه Vercel: **Settings → Environment Variables**
۲. یک متغیر جدید بساز:
   - Name: `ADMIN_PASSWORD`
   - Value: یک گذرواژه قوی که فقط خودت می‌دانی
۳. ذخیره کن و از تب **Deployments** روی آخرین deployment گزینه **Redeploy** بزن (تا متغیرها اعمال شوند).

## مرحله ۵ — استفاده

- آدرس اپ: `https://docedit-xxxx.vercel.app` (در داشبورد می‌بینی؛ می‌توانی دامنه دلخواه هم ست کنی)
- ویرایشگر همان صفحه اصلی است و برای همه باز است، ولی **انتشار فقط با گذرواژه تو** ممکن است.
- در ویرایشگر دکمه 🌐 را بزن:
  1. آدرس سرور: `https://docedit-xxxx.vercel.app`
  2. گذرواژه مدیر را وارد کن
  3. **انتشار** بزن → سند در دیتابیس ذخیره می‌شود
  4. در بخش «لینک‌های اشتراک»: تعداد مرورگر یونیک (مثلا ۵) و اجازه کامنت را تعیین کن → **ساخت لینک اختصاصی**
  5. لینک را کپی کن و بفرست.

### رفتار لینک اختصاصی
- هر مرورگر یک شناسه یکتای تصادفی می‌گیرد (در حافظه مرورگر خواننده ذخیره می‌شود).
- اگر حداکثر را **۱** بگذاری: اولین نفری که باز کند لینک برای بقیه قفل می‌شود؛ خودش همیشه می‌تواند دوباره باز کند.
- اگر **۵** بگذاری: فقط ۵ مرورگر یونیک اول دسترسی می‌گیرند.
- از پنل انتشار می‌توانی هر لینک را **ابطال** (🗑) کنی یا تعداد استفاده‌اش را ببینی.
- خواننده‌ها فقط می‌خوانند و (اگر اجازه داده باشی) کامنت می‌گذارند؛ هیچ‌کس جز تو نمی‌تواند سند را تغییر دهد.

> ⚠️ نکته صادقانه: «مرورگر یونیک» یعنی شناسه ذخیره‌شده در مرورگر. اگر کسی حافظه مرورگرش را پاک کند یا حالت ناشناس باز کند، یک «مرورگر جدید» حساب می‌شود و یکی از ظرفیت‌ها را می‌گیرد. برای اشتراک‌گذاری مستندات معمولی این روش کاملا کافی است.

| سرویس | سقف رایگان |

## اجرای محلی (مثل قبل)

`run-windows.bat` را اجرا کن. ویرایش و خروجی‌گرفتن کاملا آفلاین-محور است؛ فقط انتشار/اشتراک به سرور Vercel وصل می‌شود (آدرس سرور را در پنل 🌐 وارد کن).
